Smart Delivery and Route Optimization System - Backend Progress
Project Overview

This is a Dockerized microservices-based backend system for a Smart Delivery and Route Optimization platform. The architecture follows distributed system principles using independent services connected through an API Gateway. The platform is being developed as a hackathon MVP using free/open-source technologies and MongoDB Atlas as the cloud database.

7. Route Optimization Service ✅ — Port 5005
Features
Multi-stop route optimization using Nearest-Neighbour TSP heuristic
Haversine formula for accurate GPS distance calculation
ETA estimation (30 km/h average speed + 10 min per stop buffer)
Distance calculator between any two coordinates
JWT-protected endpoints
Pure Node.js — no external optimization library needed

Algorithm
Nearest-Neighbour TSP (O(n²)):
  Start at driver/warehouse origin
  At each step, visit the closest unvisited stop
  Returns ordered stops, per-segment distances, total distance, ETA

APIs
Method  Endpoint              Access     Description
POST    /api/routes/optimize  Protected  Optimize multi-stop delivery route
POST    /api/routes/distance  Protected  Calculate distance + ETA between 2 points
GET     /api/routes/test      Public     Health check

Request format for /optimize:
{
  "start": { "lat": 28.6139, "lon": 77.2090 },
  "stops": [
    { "id": "order1", "lat": 28.7041, "lon": 77.1025, "address": "Stop A" },
    { "id": "order2", "lat": 28.5355, "lon": 77.3910, "address": "Stop B" }
  ]
}

Response includes:
  orderedStops    — stops reordered by nearest-neighbour
  segments        — per-leg distances
  totalDistanceKm — total route distance
  estimatedTimeFormatted — human-readable ETA (e.g. "1h 45m")

8. RBAC Middleware ✅ — All Services
Each service (order, tracking, route-optimization) has its own
protect + authorize middleware that verifies JWT locally using JWT_SECRET.
No inter-service auth calls — stateless and fast.

protect  — verifies Bearer token, attaches { id, role } to req.user
authorize(...roles) — restricts route to specified roles, returns 403 otherwise
